# Ollama Training Data

This directory contains training data for fine-tuning the local Ollama model.

## File Descriptions

| File | Description |
|------|-------------|
| `candidates.jsonl` | Auto-extracted training candidates from eval results + Claude-generated gold responses. Not yet curated. |
| `approved.jsonl` | Curated, human-approved training examples. Used as the source for train/test/valid splits. |
| `train.jsonl` | Training split (80%). MLX-clean format (messages only, no metadata). |
| `test.jsonl` | Test split (10%). Used during training to monitor generalization. |
| `valid.jsonl` | Validation split (10%). Held out for final evaluation. |

## JSONL Format

### Candidates & Approved (with metadata)

Each line is a JSON object:

```json
{"messages": [{"role": "system", "content": "..."}, {"role": "user", "content": "..."}, {"role": "assistant", "content": "..."}], "metadata": {"taskId": "tool-use-001", "category": "tool-use", "model": "qwen2.5-coder:32b", "score": 5, "source": "eval-extract"}}
```

### Train/Test/Valid (MLX-clean)

Stripped of metadata — only the `messages` array:

```json
{"messages": [{"role": "system", "content": "..."}, {"role": "user", "content": "..."}, {"role": "assistant", "content": "..."}]}
```

This is the MLX `chat` format used by `mlx_lm.lora --train`.

## Data Sources

| Source | Description | How |
|--------|-------------|-----|
| `eval-extract` | High-scoring responses (4-5) from Ollama eval runs | `data-collector.js extract --min-score 4` |
| `claude-gold` | Ideal responses generated by Claude for tasks where Ollama scored poorly (1-2) | `data-collector.js generate-gold --max-score 2` |
| `manual` | Hand-crafted examples added directly to approved.jsonl | Edit file directly |

## Curation Process

1. **Extract** candidates from eval results:
   ```
   node scripts/ollama-eval/data-collector.js extract --min-score 4
   ```
2. **Generate gold** responses for weak areas:
   ```
   node scripts/ollama-eval/data-collector.js generate-gold --max-score 2
   ```
3. **Review** candidates interactively:
   ```
   node scripts/ollama-eval/data-curator.js
   ```
4. **Split** approved data for training:
   ```
   node scripts/ollama-eval/data-collector.js split
   ```

## Quality Bar

A training example should:

- Have a clear, well-formed prompt
- Contain a response that is correct, concise, and follows instructions
- Score 4+ on the eval rubric (or be a Claude-generated gold response)
- Not contain harmful, incorrect, or low-quality content
- Include a system prompt that sets appropriate context

## Adding Examples Manually

Append to `approved.jsonl` with this format:

```json
{"messages": [{"role": "system", "content": "You are a coding assistant."}, {"role": "user", "content": "Your question here"}, {"role": "assistant", "content": "The ideal response"}], "metadata": {"taskId": "manual-001", "category": "custom", "model": "manual", "score": 5, "source": "manual"}}
```

Then run `node scripts/ollama-eval/data-collector.js split` to regenerate the train/test/valid splits.
